source_transactions.csv
Ground-truth source table export from PostgreSQL
25 rows
Columns: 6
Schema: fixed
Format: CSV

        

          What this file is: A 25-row export from the src.transactions PostgreSQL table — the upstream source of truth. Columns are transaction_id, customer_id, account_balance, amount, transaction_date, status.


          What the agent checks: The SQL Validator compares this row count and null rates against the target table. If the target has significantly fewer rows or more nulls, it flags a data quality issue.


          This file never changes across scenarios — it's the baseline the agent measures everything else against.

          What this file is: The tgt.transactions table after the Spark transformation job. This is what actually landed in the data warehouse.


          What the agent checks: Row count vs source (threshold: 5% drop), per-column null rates (threshold: 5%), and duplicate transaction_ids.


          Try the failure modes: Row Drop loads 15 rows (40% missing). Null Spike loads 25 rows with 9 empty customer_ids (36% null rate). Upload your own CSV with the same columns to test against real data.

          What this file is: The raw stdout log from the Spark executor. In production this comes from YARN / Databricks cluster logs.


          What the agent checks: Scans for ERROR and WARN lines, extracts error messages, and reads Kafka consumer lag numbers using regex.


          Try the failure modes: Row Drop log shows partition skew and deduplication errors. Schema Drift shows a missing-column error. Null Spike shows a NullPointerException. Kafka Lag shows lag growing past 15,000 messages.

          What this file is: JSON maps of column_name → data_type for both src.transactions and tgt.transactions. In production, these are fetched from information_schema.columns.


          What the agent checks: Compares source and target column sets. Detects renames (same type, different name), removals, additions, and type changes.


          Try Schema Drift: The drifted target has account_balance renamed to bal — the exact kind of silent breakage a Flyway migration can cause in the wrong environment.
